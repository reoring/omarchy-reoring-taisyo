# CSKK Ascii passthrough (Vim/games)

fcitx5-cskk's `@` (Ascii) input mode normally commits ASCII characters directly.
Some apps rely on raw key events (Obsidian Vim mode, terminal/TUI keybinds, games) and behave incorrectly if the IME consumes the key event and only sends committed text.

This repo configures CSKK so that when CSKK is in Ascii mode, most keys are passed through as key events (`PassthroughKeyEvent`).

## Implementation in this repo

- Fcitx5 addon config: `home/.config/fcitx5/conf/fcitx5-cskk`
  - installed to `~/.config/fcitx5/conf/fcitx5-cskk`
  - sets `Rule=passthrough_ascii`
- libcskk rules: `~/.local/share/libcskk/rules/`
  - created/updated by `apply.sh`
  - `apply.sh` generates `~/.local/share/libcskk/rules/passthrough_ascii/rule.toml` from the system default rule and replaces only the `[direct.ascii]` section.

## User-facing behavior

- Enter CSKK Ascii mode from Hiragana by pressing `l` (this is part of the underlying libcskk rule).
- While in CSKK Ascii mode:
  - letters/digits/punctuation are delivered to the application as key events
  - Vim normal/view mode keys like `$`, `:`, `^`, `%`, etc work as commands instead of being inserted as text
  - common Ctrl shortcuts are also passed through (e.g. `Ctrl-a`..`Ctrl-z`, `Ctrl-[`, `Ctrl-]`, `Ctrl-\`, `Ctrl-^`, `Ctrl-_`)
- Exit CSKK Ascii mode with `Ctrl-j` (returns to Hiragana).

## Why there are multiple Shift spellings

Depending on the frontend/backends, a shifted symbol may be represented differently.
For example `$` may appear as:

- `dollar`
- `(shift 4)`
- `(shift dollar)` (keysym is already shifted, but Shift modifier is still set)

The rule covers all forms for reliable passthrough.

## Apply / reload

After running `bash ./apply.sh`, restart fcitx5:

```bash
systemctl --user restart app-org.fcitx.Fcitx5@autostart.service
```

## Troubleshooting

- If a specific key still does not behave like a keybind:
  1. Run `wev` and press the key; note the `sym: ...` value.
  2. Add a matching entry to `[direct.ascii]` in the generated rule with `"PassthroughKeyEvent"`.
     - For a permanent change, update the `ascii_section` template in `apply.sh` (the rule file is regenerated by `apply.sh`).
  3. Restart fcitx5.

- To see what key events fcitx receives (useful when debugging):
  1. Stop the systemd service:

     ```bash
     systemctl --user stop app-org.fcitx.Fcitx5@autostart.service
     ```

  2. Run fcitx5 in foreground with key tracing:

     ```bash
     fcitx5 -D --verbose 'default=4,key_trace=5'
     ```

## Arch note: libcskk shared library path

On Arch, `cskk-git` installs `libcskk.so` under `/usr/lib/cskk/`.
This repo ships a systemd user override to make it visible to `fcitx5-cskk`:

- `home/.config/systemd/user/app-org.fcitx.Fcitx5@autostart.service.d/override.conf`
  - `Environment=LD_LIBRARY_PATH=/usr/lib/cskk`
